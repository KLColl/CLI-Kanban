## Можливості

| Категорія | Функції |
|-----------|---------|
| 📋 **Дошка** | Перегляд задач у трьох колонках: Todo / In Progress / Done |
| ➕ **Задачі** | Додавання, видалення, переміщення, редагування |
| ✅ **Підзадачі** | Чеклісти з прогресом `2/3` для кожної задачі |
| 🏷️ **Теги** | Мітки на задачах, фільтрація за тегом |
| 🔺 **Пріоритети** | `low` / `medium` / `high` з кольоровим виділенням |
| 📅 **Дедлайни** | Встановлення та виділення прострочених задач |
| 🔍 **Пошук** | За текстом, пріоритетом, тегом, дедлайном |
| 📦 **Архів** | Перенесення виконаних задач окремо |
| 📊 **Статистика** | Розподіл за статусами / пріоритетами, % виконання |
| 📤 **Експорт** | Збереження дошки у формати CSV та Markdown з історією дій |

## Запуск

**Вимоги:** Python 3.10+

```bash
# 1. Клонувати репозиторій
git clone https://github.com/KLColl/CLI-Kanban.git
cd CLI-Kanban

# 2. Створити та активувати віртуальне середовище
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS / Linux

# 3. Запустити
python main.py
```

Після запуску відкривається головне меню з трьома розділами:

```
========== KANBAN-МЕНЕДЖЕР ЗАДАЧ ==========
1. Дошка та задачі
2. Пошук і фільтри
3. Статистика
4. Експорт 
0. Вийти
```

## Формат даних

Задачі зберігаються у `board.json` як масив об'єктів:

```json
{
  "id": 1,
  "title": "Назва задачі",
  "description": "Опис",
  "status": "todo",
  "priority": "high",
  "deadline": "25.12.2026",
  "created_at": "19.06.2026",
  "tags": ["feature", "backend"],
  "subtasks": [
    { "id": 1, "title": "Підзадача", "done": false }
  ],
  "history": [
    { "timestamp": "19.06.2026 10:00", "action": "Задачу створено зі статусом Todo" }
  ]
}
```

Виконані задачі при архівуванні переносяться у `archive.json` з тією самою структурою.

## Експорт даних
**Програма підтримує вивантаження поточної дошки у зовнішні файли за допомогою окремого модуля `exporter.py`:**

**1. CSV (.csv):** Формує таблицю з людиночитабельними заголовками українською мовою. 
Містить повну інформацію про задачі, включаючи статус прострочення та історію змін 
(записи історії розділені переносом рядка всередині клітинки для зручного перегляду в Excel через функцію *Wrap Text*). 
Для коректного відображення кирилиці використовується кодування `utf-8-sig`.


**2. Markdown (.md):** Генерує структурований звіт з блоком загальної статистики дошки 
(% виконання, кількість прострочених задач). Задачі групуються за колонками-статусами у вигляді таблиць. 
Прострочені дедлайни підсвічуються символом `⚠️`, теги оформлені як код, а історія дій відображається компактними списками. 
Усі системні символи автоматично екрануються, щоб зберегти цілісність таблиці.

## Тестування

```bash
python -m pytest test_kanban.py -v
```

Усі тести успішно проходять.


**Покриття тестами:**

| Модуль | Що перевіряється |
|--------|-----------------|
| `add_task` | Створення, унікальність ID, валідація назви / пріоритету / дедлайну |
| `move_task` | Переміщення між статусами, запис в `history`, граничні випадки |
| `filter` / `search` | Фільтр за пріоритетом і тегом, текстовий пошук, сортування |
| `is_overdue` | Прострочені / майбутні / виконані / без дедлайну |
| `subtasks` | Додавання, toggle, прогрес підзадач |
| `get_statistics` | Підрахунок по статусах, completion rate, порожня дошка |
| `parse_tags` | Парсинг рядка, дедуплікація, список на вході |
